# 2D-RUNNER-GAME

`2D-RUNNER-GAME`은 Unity로 제작한 2D 리듬 러너 게임입니다.

플레이어는 신전을 탐험하는 용사가 되어, 신전을 수호하는 골렘의 공격을 피하면서 시간의 룬 조각을 모아야 합니다.  
게임은 약 70초 길이의 음악을 기준으로 진행되며, 플레이어는 `Space` 키 하나로 상/하 레인을 전환해 음악 타이밍에 맞춰 아이템을 획득하고 장애물을 회피합니다.

이 프로젝트는 단순한 조작 방식의 러너 게임에 리듬 게임 요소를 결합한 것이 특징입니다.  
복잡한 조작보다는 정확한 타이밍 판단과 반복 플레이의 재미에 집중했으며, 예전 플래시 게임처럼 가볍지만 계속 도전하게 되는 중독성을 목표로 제작했습니다.

전체적인 분위기는 고전 RPG와 픽셀 아트 스타일을 기반으로 구성했습니다.  
UI는 모험가 길드의 의뢰서 같은 느낌을 참고했으며, HP, 아이템, 플레이어, 보스 등 주요 에셋도 픽셀 아트 기반으로 통일했습니다. 또한 게임 진행도에 따라 배경과 보스의 색상이 변화하여, 밝은 낮 분위기에서 점점 어두운 밤 분위기로 전환되는 연출을 추가했습니다.

## 주요 기능

### 위/아래 위치 전환 조작

플레이어는 `Space` 키를 눌러 위쪽과 아래쪽 위치를 전환할 수 있습니다.
이 조작을 통해 등장하는 아이템을 획득하거나 장애물을 회피합니다.

### 아이템 획득 및 점수 증가

게임 중 등장하는 아이템과 충돌하면 해당 아이템이 제거되고 점수가 증가합니다.
현재 점수는 게임 UI에 표시됩니다.

### 장애물 충돌 및 체력 감소

장애물과 충돌하면 플레이어의 체력이 감소합니다.
체력은 하트 UI로 표시되며, 체력이 모두 소진되면 게임이 종료됩니다.

### 시간 기반 게임 진행

게임은 시작 후 시간이 흐르면서 단계적으로 변화합니다.

* 초반에는 일반 노트와 장애물이 등장합니다.
* 일정 시간이 지나면 보스가 등장합니다.
* 이후 보스가 강화 상태로 전환되며 배경 분위기와 공격 확률이 변화합니다.
* 제한 시간이 지나거나 체력이 모두 소진되면 게임이 종료됩니다.

### 보스 패턴

보스는 게임 진행 중 등장하여 플레이어의 위치에 대응하는 장애물을 추가로 생성합니다.
시간이 지날수록 보스의 공격 확률이 높아져 후반부 난이도가 상승합니다.

### 타이틀 / 게임 오버 UI

게임 시작 전에는 타이틀 UI가 표시됩니다.
`Enter` 키를 누르면 게임이 시작되며, 게임 오버 시 다시 타이틀 UI가 활성화되고 최종 점수가 표시됩니다.

## 구현 상세

### 전체 게임 흐름 관리

`GameManager`는 게임의 전체 상태를 관리합니다.

* 타이틀 상태와 플레이 상태 전환
* 게임 시작 처리
* 경과 시간 계산
* 체력 UI 갱신
* 점수 UI 갱신
* 게임 오버 처리
* 플레이어와 오브젝트 생성 시스템 초기화 및 종료

싱글톤 형태의 `GameManager.Instance`를 사용하여 다른 스크립트에서 점수, 체력, 게임 오버 처리를 호출할 수 있도록 구성했습니다.

### 플레이어 제어

`Player` 스크립트는 플레이어의 입력, 애니메이션, 충돌 처리를 담당합니다.

* `Space` 입력을 통해 위/아래 상태 전환
* 상태에 따라 `ASCEND`, `DESCEND` 애니메이션 트리거 실행
* 아이템 충돌 시 점수 증가
* 장애물 충돌 시 체력 감소
* 피격 시 파티클 효과 재생
* 게임 시작/종료 시 업데이트 활성화 여부 제어

플레이어의 상태는 `onUpside` 값으로 관리되며, 게임이 진행 중일 때만 입력과 충돌 처리가 동작하도록 구성했습니다.

### 노트 및 장애물 생성 시스템

`Factory` 스크립트는 시간 기반으로 노트와 장애물을 생성합니다.

* JSON 파일에 저장된 노트 타이밍 데이터를 읽어옴
* 경과 시간이 특정 노트 타이밍에 도달하면 오브젝트 생성
* 노트 인덱스에 따라 위/아래 위치를 번갈아 사용
* 게임 시작 시 오디오를 예약 재생
* 게임 종료 시 오디오 및 생성 로직 정지

이를 통해 고정된 시간표에 따라 오브젝트가 생성되는 리듬 게임식 진행 구조를 구현했습니다.

### 보스 시스템

`Boss` 스크립트는 보스의 등장, 상태 변화, 공격 패턴을 담당합니다.

* 일정 시간이 지나면 보스가 깨어나며 등장
* 보스가 분노 상태가 되면 색상과 배경색 변경
* 공격 확률을 조정하여 후반부 난이도 상승
* 일정 조건에 따라 위/아래 장애물 생성
* 보스 사망 시 게임 오버 처리

보스 공격은 코루틴을 사용하여 애니메이션 트리거 이후 일정 시간 뒤에 장애물이 생성되도록 구현했습니다.

### 배경 및 지면 스크롤

`Ground` 스크립트는 여러 개의 지면 세그먼트를 왼쪽으로 이동시키고, 화면 밖으로 나간 세그먼트를 다시 오른쪽 끝으로 배치합니다.

이를 통해 제한된 개수의 지면 오브젝트만으로도 무한히 이어지는 러너 게임 배경을 구현했습니다.

### 오브젝트 이동 및 제거

`Move` 스크립트는 아이템, 장애물, 노트 등의 오브젝트를 왼쪽으로 이동시킵니다.

* 매 프레임 `speed` 값에 따라 왼쪽으로 이동
* 카메라의 왼쪽 경계를 벗어나면 자동 제거
* 화면 밖 오브젝트가 계속 누적되지 않도록 관리

### 노트 데이터 제작 도구

`NoteMaker` 스크립트는 게임 내 노트 타이밍 데이터를 제작하기 위한 보조 도구입니다.

* 오디오를 재생하면서 특정 키 입력 시 현재 시간을 기록
* 기록된 시간 목록을 JSON 형식으로 저장
* 저장된 JSON 데이터를 `Factory`에서 읽어 게임 오브젝트 생성 타이밍으로 사용

이를 통해 수작업으로 노트 타이밍을 기록하고 게임 진행 데이터로 활용할 수 있도록 구성했습니다.
